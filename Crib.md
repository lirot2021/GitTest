`
## Отмена индексации
`
git restore --staged <file>
`

## Отмена изменений файла
`
git restore <file>
`

# КОМАНДА ПРОСМОТРА ИСТОРИИ КОМИТОВ

`git log`

Аргументы:
- `-p` Показывает патч внесённый в каждый комит
- `--stat` печатает под каждым из коммитов список и количество изменённых файлов, а также сколько строк в каждом из файлов было добавлено и удалено. В конце можно увидеть суммарную таблицу изменений
- `--pretty` меняет формат вывода
- `git log --pretty=format: опции в кавычках` позволяет указать формат для вывода информации. Список опций[](https://git-scm.com/book/ru/v2/%D0%9E%D1%81%D0%BD%D0%BE%D0%B2%D1%8B-Git-%D0%9F%D1%80%D0%BE%D1%81%D0%BC%D0%BE%D1%82%D1%80-%D0%B8%D1%81%D1%82%D0%BE%D1%80%D0%B8%D0%B8-%D0%BA%D0%BE%D0%BC%D0%BC%D0%B8%D1%82%D0%BE%D0%B2#rpretty_format 
)
- `--graph` небольшой граф в формате ASCII, который показывает текущую ветку и историю слияний
- `--since=` Ограничения по дате
  
`git log --since=2.weeks` за последние две недели

`git log --since=2008-01-15` с 15 января 2008

`git log --since=2 years 1 day 3 minutes ago` за 2 года 1 день и 3 минуты
- `-S строка` Ищет комиты которые добавляли или удаляли строку
- `git log -- path/to/file` комиты связанные с определённым файлом
